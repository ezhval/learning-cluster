# Day_05 — Моделирование требований: DFD, Swimlane, артефакты и состояния

В этом дне фокус на визуальных моделях, которые помогают согласовать требования:  
**DFD**, **swimlane-диаграммы**, их расширение артефактами, **диаграммы состояний** и **таблицы состояний**.

Модели создаются по двум кейсам:
- Barbershop Appointment Project (BAP),
- Delivery Order System (DOS).

**Фокус дня** — научиться описывать логику системы через потоки данных, роли, шаги процессов и жизненный цикл объектов.

---

## Упражнение 00 — DFD Level 0

**Задача:**  
Построить контекстную диаграмму потоков данных (DFD Level 0) для выбранного кейса.

Нужно:

- указать цель диаграммы и As-Is/To-Be;
- отметить внешних акторов (в прямоугольниках);
- изобразить процессы верхнего уровня (овалы) и их нумерацию;
- показать хранилища данных (параллельные линии);
- прописать на стрелках содержимое информационных потоков;
- не отображать последовательность действий — только движение данных.

## Упражнение 02 — Расширение swimlane-диаграммы артефактами

**Задача:**  
Добавить к диаграмме артефакты, которые создают роли.

Нужно:

- скопировать диаграмму из упражнения 01 и переименовать файл;
- указать цель и As-Is/To-Be;
- в каждой дорожке отметить документы, записи, обращения и другие артефакты, создаваемые ролью;
- связать артефакты с шагами пунктиром;
- при необходимости группировать артефакты.

**Результат:**

- `ex02_DOS_adswd.jpg`
