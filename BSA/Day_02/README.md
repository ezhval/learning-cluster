# Day_02 — Требования, контекст и модели As-Is / To-Be

В этом дне начинаем формализовывать требования и структуру будущей системы.  
Работа идёт на двух кейсах:

- Barbershop Appointment Project (BAP) — система онлайн-записи в барбершоп;  
- Delivery Order System (DOS) — система курьерской доставки заказов.

**Фокус дня** — зафиксировать текущее состояние (As-Is), наметить целевое (To-Be), описать контекст системы и информационные потоки, а также синхронизировать артефакты (таблицы ролей, глоссарий, реестры стейкхолдеров).

---

- для каждой роли перечислить её действия в текущих процессах;
- зафиксировать, какие проблемы и ограничения возникают при таком способе работы.

**Результат:**
