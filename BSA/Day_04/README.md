# Day_04 — Предметная область и информационная модель

В этом дне фокус на формализации предметной области через сущности и данные.  
Работа ведётся по двум кейсам:

- **Barbershop Appointment Project (BAP)** — система онлайн-записи в барбершоп;  
- **Delivery Order System (DOS)** — система курьерской доставки заказов.

**Фокус дня** — выделить сущности предметной области, проверить полноту действий по CRUD, оформить данные в виде словаря и собрать логическую модель данных (ER-диаграмму).

---

## Упражнение 00 — Идентификация сущностей

**Задача:**  
По описаниям задач BAP и DOS выделить сущности, с которыми будет работать система.

Нужно:

- пройтись по тексту задачи и зафиксировать кандидатов в сущности (по подходу «существительное + действие»);
- выбрать из кандидатов те сущности, которые реально будут представлены в данных системы (объекты, события и т.п.);
- для каждой сущности:
  - задать понятное имя и кратко сформулировать назначение;
  - определить ключевые атрибуты (что отличает один экземпляр от другого).

**Результат:**

- `ex00_подготовка.docx` 
- `ex00_BAP_entity.xlsx`  
- `ex00_DOS_entity.xlsx`

## Упражнение 01 — Проверка сущностей по CRUD (только BAP)

**Задача:**  
Для кейса **BAP** проверить полноту действий над сущностями через матрицу CRUD.

Нужно:

- для каждой сущности указать, какие роли и в каких сценариях:
  - создают записи (Create),
  - просматривают/читают (Read),
  - изменяют (Update),
  - удаляют или выводят из активного оборота (Delete/архивируют);
- если по условию задачи неочевидно, кто выполняет операцию или можно ли её делать, — зафиксировать гипотезу или явно пометить, что требуется уточнение у стейкхолдеров.

**Результат:**

- `ex01_BAP_crud.xlsx`

## Упражнение 02 — Словарь данных

**Задача:**  
Для обоих кейсов собрать словарь данных, который описывает сущности и их атрибуты уже не на уровне «слов», а на уровне данных.

Нужно:

- для каждой сущности и её атрибутов указать, как минимум:
  - наименование сущности / атрибута;
  - краткое бизнес-описание;
  - условное обозначение / мнемонику (если используется);
  - тип данных (число, строка, дата и т.п.);
  - обязательность (обязательное поле или нет);
- при необходимости зафиксировать ограничения (формат, допустимые значения и т.д.).

**Результат:**

- `ex02_BAP_dict.xlsx`  
- `ex02_DOS_dict.xlsx`

## Упражнение 03 — Логическая модель данных (ER-диаграмма)

**Задача:**  
Построить логическую модель данных в формате ER-диаграммы для каждого кейса.

Нужно:

- включить в ER-диаграмму:
  - сущности, выделенные в упражнении 00;
  - справочники / словари (reference data), если они нужны;
  - разрывающие таблицы для связей многие-ко-многим (M:N);
- для каждой связи:
  - указать кратность (1:1, 1:N, M:N);
  - добавить глагольные подписи, поясняющие смысл связи;
- убедиться, что модель согласована с CRUD и словарём данных.
