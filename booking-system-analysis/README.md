# Barbershop Booking System – System Analysis Project

Этот pet-проект демонстрирует подход к анализу и проектированию ИТ-систем на примере сервиса онлайн-записи в барбершоп.  
Цель проекта — показать навыки системного анализа: работу с архитектурой, моделями данных, API-контрактами и процессами.

Проект выполнен в учебных целях и отражает «сквозной» путь от предметной области и данных до архитектуры микросервисов и спецификации API.

---

## Структура проекта

```
barbershop-booking-system-analysis/
│
├── architecture/       → архитектура решения и взаимодействие сервисов
├── api/                → API-контракты (OpenAPI / Swagger)
├── data-model/         → модель данных, ERD, CRUD, словарь
└── README.md           → описание проекта
```


## Что реализовано в проекте

### Архитектура решения

В проекте выполнено:

- выделение ключевых доменов предметной области barbershop booking;
- разбиение на микросервисы (Client, Booking, Master, Catalog, Payment, Notification);
- описание ответственности каждого сервиса;
- определение вариантов интеграции (REST, event-driven, async);
- фиксация точек взаимодействия через API.

Документы находятся в папке **`/architecture`**.

---

### Модель данных

В рамках моделирования данных подготовлены:

- ER-диаграмма;
- перечень сущностей и атрибутов;
- CRUD-матрица;
- словарь предметной области (glossary).

Материалы в папке **`/data-model`**.

---

### API Design (OpenAPI / Swagger)

API-контракты разработаны для основных сервисов системы:

- **Booking Service**
- Client Service
- Master Service
- Catalog Service
- Payment Service

Формат: OpenAPI 3.0.  
Файлы находятся в **`/api`**.

---

### Цель проекта

Проект направлен на демонстрацию навыков системного аналитика:

- анализ предметной области и выделение доменов;
- проектирование микросервисной архитектуры;
- разработка API-контрактов;

- описание процессов и взаимодействий;
